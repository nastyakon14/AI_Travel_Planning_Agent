# Интеграции Tools / APIs

## Роль

Получение данных о билетах и отелях через [Travelpayouts](https://www.travelpayouts.com/) (Aviasales API) и обогащение маршрута через LLM (достопримечательности).

## Контракты (сокращённые)

- **`search_flights`** — после известных origin/destination и дат: список с `flight_id`, `airline`, `price_eur`, `time_dep`, `time_arr`, …
- **`search_hotels`** — `hotel_id`, `name`, `price_per_night`, `rating`, …

Ответы обрезаются до топ-N (по умолчанию 5) полей, лишние поля API отбрасываются.

## Побочные эффекты

Интеграции в режиме **read-only**: бронирований и списаний нет, вызовы идемпотентны для ретраев графа.

## Ошибки и таймауты

- HTTP к внешним API и LLM ограничены таймаутами в клиентах (`LLM_TIMEOUT_SEC` и настройки `requests` в модулях aviatickets/hotels).
- При ошибке тул возвращает структуру с `error` / пустыми списками и подсказкой продолжить с ориентировочными значениями (см. `agent_tools.py`).
