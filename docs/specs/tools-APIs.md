# Интеграции Tools / APIs

## Роль
Получение реальных данных о билетах и отелях для расчета бюджета.

## Контракты (Схемы данных)
- **`search_flights(origin: str, dest: str, date: str)`** 
  - *Возвращает:* Список JSON `[{flight_id, airline, price_eur, time_dep, time_arr}]`.
- **`search_hotels(city: str, dates: str)`**
  - *Возвращает:* Список JSON `[{hotel_id, name, price_per_night, rating}]`.

## Side effects (Сайд-эффекты)
Интеграции работают строго в режиме **Read-Only**. Инструменты не совершают транзакций и бронирований, поэтому их вызов абсолютно безопасен и идемпотентен (можно вызывать многократно при ретраях графа).

## Ошибки и Timeout (Инфраструктурный контроль)
- **Таймаут:** Жесткий таймаут на вызов любого внешнего API — **20 секунд**.
- **Обработка падений:** Если API падает (5xx) или таймаутит, выбрасывается контролируемое исключение, которое перехватывается оберткой тула (Tool Node).

## Fallback / Защита контекста
- **Ответ при ошибке:** Обертка тула возвращает в LLM строку: `{"error": "API Timeout. Use estimated mock data: Flight ~150 EUR"}`. Агент промптирован продолжать работу с мок-данными.
- **Защита от OOM (Out of Memory) контекста:** API отелей часто возвращает огромные JSON с сотнями полей (картинки, отзывы). Перед возвратом в агента данные **жестко фильтруются** (оставляем только `id`, `name`, `price`) и ограничиваются **топ-5** самыми дешевыми вариантами.
