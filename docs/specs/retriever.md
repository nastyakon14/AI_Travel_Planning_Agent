# Модуль Retrieval (RAG)

## Роль
Обогащение контекста локальными достопримечательностями и правилами (например, визовыми), чтобы снизить галлюцинации LLM при планировании маршрута на каждый день.

## Источники и Индекс
- **Источники:** Статические датасеты (сборки из Wikipedia / WikiVoyage) по топ-50 популярным туристическим городам.
- **База данных:** Qdrant или ChromaDB (in-memory для локального запуска, persistent volume для демо).
- **Модель эмбеддингов:** `text-embedding-3-small` (OpenAI).
- **Chunking strategy:** Разбиение по смысловым блокам (название достопримечательности + краткое описание + примерное время на посещение). Размер чанка: ~300-500 токенов.

## Поиск (Retrieval)
- **Метод:** Cosine Similarity.
- **Настройки:** `Top-K = 5` объектов для каждого дня поездки.

## Ограничения и Fallbacks
- **Отсутствие данных:** Если город пользователя отсутствует в векторной базе (score поиска ниже `0.6`), модуль возвращает пустой список. 
- **Fallback:** Агент переключается на внутренние знания LLM (parametric memory), но в итоговом плане ставится системный дисклеймер: *"Информация о достопримечательностях сгенерирована ИИ и может быть неточной, так как город вне нашей основной обновляемой базы"*.
