# Спецификация: Observability & Evals

## Логи и Трейсы (Observability)
- **LangSmith:** Обязательная интеграция. Используется для визуализации графа агента, просмотра входов/выходов каждого узла и дебага "мыслей" агента (почему он вызвал именно этот инструмент с этими параметрами).
- **Системные логи (Python `logging`):** Логируются факты старта сессии, ошибки таймаутов API и факты срабатывания Guardrails. API ключи и персональные данные маскируются (redaction).

## Метрики
- `agent_run_duration_seconds`: end-to-end время ответа пользователю.
- `llm_token_usage_total`: расход токенов на одну сессию (для контроля unit-экономики проекта).
- `tool_failure_rate`: процент упавших/отбивших по таймауту внешних API.

## Evals (Оценка качества на этапе CI/CD)
Разрабатывается тестовый датасет из 20 сценариев (например, "Невозможный бюджет", "Сложный маршрут на 10 дней", "Город без аэропорта").
Для проверки используется паттерн **LLM-as-a-Judge** (оценка одного LLM с помощью другого LLM-судьи). 

**Критерии приемки теста:**
1. **Бюджет:** Сумма цен в итоговом ответе строго $\le$ заявленному бюджету (если не сработал лимит ретраев). Проверяется математически парсером.
2. **Форматирование:** Ответ является валидным Markdown-документом с разбивкой по дням.
3. **Отсутствие галлюцинаций:** Использованы только те отели и билеты, которые вернули инструменты (проверяется сверкой ID в финальном ответе и ID в State графа).
