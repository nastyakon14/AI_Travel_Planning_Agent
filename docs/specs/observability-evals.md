# Наблюдаемость и оценки качества

## Что есть в коде

- **Логи:** стандартный вывод процесса Streamlit; чувствительные данные в логи не добавлять.

Интеграция **LangSmith** в репозитории не подключена; при необходимости настройте по [документации LangChain](https://python.langchain.com/docs/langsmith/).

## Идеи для evals (CI/CD)

Набор сценариев (крайние бюджеты, длинные маршруты, неоднозначные города) и проверки:

- оценка бюджета согласована с guardrail;
- ответ — валидный Markdown с разбивкой по дням;
- ссылки на реальные `hotel_id` / `flight_id` из ответов тулов (если требуется строгая верификация).
